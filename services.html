Real SA risk: Eskom spike corrupts NAS, or M365 sync deletes SharePoint folder – Microsoft only keeps 93 days, no default backup.

Timeline

  1. Day 1: Audit what is / isn't backed up: M365, Sage, files, devices
  2. Day 2: Set immutable M365 backup + local NAS replica with UPS
  3. Day 3: Restore test drill + runbook doc: who calls who
  4. Monthly: Auto restore test, alert if fails, report shows last success

What's included (10 items)

  • •M365 backup immutable: Mail, Teams, SharePoint, OneDrive – 1yr, not 93 days
  • •Local NAS with snapshots – survives ISP outage, instant restore 10x faster
  • •3-2-1 rule: 3 copies, 2 local, 1 offline immutable cloud (Wasabi/AWS)
  • •Ransomware-proof: Backups can't be encrypted/deleted by domain admin
  • •Daily auto test: We restore 3 random files + show screenshot in report
  • •Sage Pastel / Xero backup: DB dumps hourly + off-site
  • •Device seed backup: Intune Autopilot + OneDrive Known Folder Move so PC loss != data loss
  • •Loadshedding-safe writes: UPS + journaling so NAS not corrupt on power cut
  • •15-min restore SLA: File level; 4hr server/SharePoint level
  • •POPIA incident pack: Restore proof, logs for regulator/insurer
